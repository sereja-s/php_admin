Проверка версии PHP: OS -> настройки -> модули

Конфигурация PHP: OS -> Дополнительно -> Конфигурация -> версия PHP (здесь: PHP_7.4) 
-> в файле конфигурации (для Xdebug v2.x.x): раскомментируем строчки: zend_extension = xdebug (убрать ; перед строчкой)
																  xdebug.default_enable = 1
																  xdebug.remote_enable  = 1
																  xdebug.remote_autostart = 1
																  xdebug.remote_port = 9000
 далее сохраняем файл конфигурации и можем его закрыть -> перезапускаем OS
 (Проверка версии: OS -> Дополнительно -> PHP информация (в браузере откроется страница с адресом: http://127.0.0.1/openserver/phpinfo.php)) Здесь под описанием версии PHP указана  информация о версии Xdebug (здесь: Xdebug v2.9.4)

1- в VS Code нажимаем на кнопке: Расширения (левая колонка меню)

2- в поиске вводим Xdebug -> выбираем расширение: PHP Debug и устанавливаем его

3- Настроим debugдля проекта: в VS Code нажимаем на кнопке: Запуск и отладка (левая колонка меню) -> нажимаем на ссылку: создайте файл launch.json -> в поисковой строке выбираем из предлагаемых файлов: PHP (будет создан конфигурационный файл) -> убедимся что "port": 9000 -> сохраняем файл (теперь можно его закрыть) -> перезапускаем VS Code чтобы применились настройки (на боковой панели слева (вверху) появляется кнопка (стрелочка): Listen for Xdebug) -> запускаем OS -> на боковой панели справа (вверху) нажимаем на кнопку (стрелочку): Listen for Xdebug -> (внизу появится оранжевая полоса, а вверху по центру - меню отладки, т.е. отладка запущена)

4- левой кнопкой мыши в нужном месте ставим точку останова -> запускаем наш проект (откроется вкладка браузера (если она была открыта, нажимаем F5- обновить страницу -> )) -> на боковой панели (вкладка: Запуск и отладка) будет покаана вся информация

5- дальше по коду двигаемся клавишей F10
